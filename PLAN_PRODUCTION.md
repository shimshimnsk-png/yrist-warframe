# План разработки продакшен-версии «Социальный юрист»

> Разработчик: 1 человек + Claude Code
> Общий срок: **6–8 недель** (при работе 4–6 часов/день)
> Старт: после утверждения wireframe клиенткой

---

## Фаза 0. Продажа wireframe клиентке (1–3 дня)

**Что делать:**
- Показать клиентке готовый wireframe по ссылке
- Собрать обратную связь: что нравится, что изменить, чего не хватает
- Согласовать объём работ и цену за продакшен-версию
- Подписать договор / получить предоплату

**Результат:** согласованное ТЗ + предоплата

---

## Фаза 1. Подготовка инфраструктуры (2–3 дня)

| Задача | Время | Описание |
|--------|-------|----------|
| Выбор хостинга | 2 ч | VPS (Timeweb/Beget) или PaaS (Railway/Render) |
| Настройка домена | 1 ч | Покупка домена, DNS, SSL-сертификат |
| Настройка базы данных | 3 ч | PostgreSQL (Supabase бесплатно или свой VPS) |
| Настройка CI/CD | 2 ч | GitHub Actions: тесты → сборка → деплой |
| Настройка бэкенд-проекта | 4 ч | Node.js + Express/Fastify или Next.js API Routes |

**Стоимость инфраструктуры:** ~500–1500 ₽/мес (VPS) или бесплатно (Supabase + Vercel)

---

## Фаза 2. Бэкенд и база данных (7–10 дней)

| Задача | Время | Описание |
|--------|-------|----------|
| Схема базы данных | 1 день | Таблицы: users, clients, applications, documents, services, chat_messages |
| Авторизация специалиста | 2 дня | Логин/пароль, JWT-токены, защита маршрутов |
| API заявок | 2 дня | CRUD: создание, чтение, обновление, удаление, фильтры |
| API документов | 1 день | Генерация, хранение, скачивание |
| API клиентов | 1 день | Профиль клиента, история обращений |
| API настроек | 0.5 дня | Профиль специалиста, услуги, цены |
| Генерация PDF/DOCX | 2 дня | Библиотеки: pdfkit или puppeteer (PDF), docx (DOCX) |

**Ключевое решение:** Supabase даёт БД + авторизацию + хранилище файлов из коробки — экономит ~3 дня.

---

## Фаза 3. Подключение фронтенда к бэкенду (5–7 дней)

| Задача | Время | Описание |
|--------|-------|----------|
| Авторизация на фронте | 1 день | Форма входа, хранение токена, защита маршрутов |
| Подбор пособий → сохранение | 1 день | Результаты wizard сохраняются в БД как заявка |
| Генератор документов → реальная генерация | 2 дня | Формирование настоящих PDF/DOCX по шаблону |
| Кабинет специалиста → реальные данные | 1 день | Таблица заявок из БД, фильтры, поиск |
| Карточка заявки → CRUD | 1 день | Изменение статуса, добавление комментариев |
| Настройки → сохранение | 0.5 дня | Профиль, услуги, цены в БД |

---

## Фаза 4. AI-секретарь (3–5 дней)

| Задача | Время | Описание |
|--------|-------|----------|
| Интеграция с Claude API | 1 день | Подключение API, настройка промпта |
| Скриптованная логика + AI | 2 дня | Гибрид: структурированные вопросы + свободный диалог |
| Сохранение чата в БД | 0.5 дня | История переписки привязана к заявке |
| Формирование заявки из чата | 1 день | AI извлекает данные из ответов → создаёт заявку |

**Стоимость API Claude:** ~$5–15/мес при 50–100 диалогов/мес

---

## Фаза 5. Уведомления и интеграции (2–3 дня)

| Задача | Время | Описание |
|--------|-------|----------|
| Email-уведомления | 1 день | Новая заявка → письмо специалисту (Resend/Nodemailer) |
| Telegram-бот (опц.) | 1–2 дня | Уведомления в Telegram специалисту |
| WhatsApp (опц.) | 1 день | Кнопка «Написать в WhatsApp» с предзаполненным текстом |

---

## Фаза 6. Тестирование и исправления (3–5 дней)

| Задача | Время | Описание |
|--------|-------|----------|
| Тест всех экранов на телефоне | 1 день | iPhone, Android, разные размеры |
| Тест формы подбора пособий | 0.5 дня | Все комбинации ответов дают адекватный результат |
| Тест генерации документов | 1 день | PDF/DOCX открываются, данные корректны |
| Тест AI-секретаря | 0.5 дня | Диалог ведёт к результату, заявка формируется |
| Исправление багов | 1–2 дня | Буфер на неожиданные проблемы |
| Безопасность | 0.5 дня | Проверка: XSS, SQL-инъекции, CORS, rate limiting |

---

## Фаза 7. Передача клиентке (2–3 дня)

| Задача | Время | Описание |
|--------|-------|----------|
| Деплой на продакшен | 0.5 дня | Финальная сборка, домен, HTTPS |
| Наполнение контентом | 1 день | Реальные услуги, цены, шаблоны документов клиентки |
| Обучение клиентки | 0.5 дня | Видео или созвон: как пользоваться кабинетом |
| Документация | 0.5 дня | Краткая инструкция: как менять услуги, шаблоны, цены |
| Передача доступов | 0.5 дня | Логин, пароль, домен, хостинг |

## Рекомендуемый стек для продакшена

| Слой | Технология | Почему |
|------|-----------|--------|
| Фронтенд | React + Vite + TypeScript (уже есть) | Переиспользуем wireframe |
| Бэкенд | Supabase (БД + Auth + Storage) | Экономит 3–5 дней разработки |
| AI | Claude API (Sonnet) | Русский язык, качество ответов |
| Генерация PDF | @react-pdf/renderer или puppeteer | Красивые документы |
| Генерация DOCX | docx (npm-пакет) | Редактируемые документы |
| Хостинг фронта | Vercel (бесплатно) | Автодеплой из GitHub |
| Домен | .ru через Timeweb/REG.ru | ~200 ₽/год |
| Email | Resend (бесплатно до 100/день) | Простое API |
